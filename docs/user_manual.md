## 1. 产品简介

InterviewEcho 是一个基于 RAG（检索增强生成）+ 大模型的 AI 模拟面试平台，帮助你在真实感的环境中练习技术面试并获得多维度能力评估。核心能力：

- **多岗位模拟面试**：支持 Java 后端、Web 前端、Python 算法三类岗位。
- **文本 + 语音双通道作答**：可打字，也可直接说话，由系统转写并分析表达。
- **个性化深挖**：可上传简历、粘贴 GitHub 项目链接，AI 据此定制提问。
- **代码在线练习与判题**：Hot100 ACM 模式题库，支持运行样例与正式提交判题。
- **自动化评估报告**：面试结束后生成内容 + 表达多维评分、亮点、待改进点与提升建议，并可导出 PDF。

## 2. 快速开始

1. 打开平台首页，点击进入登录页，注册并登录账号。
2. 进入仪表盘，选择一个目标岗位。
3. 在弹出的配置框中设置难度、轮次、考察方向（可选上传简历 / 项目链接），点击「确认配置并进入房间」。
4. 在面试房间中用文字或语音回答 AI 面试官的问题。
5. 完成后点击「结束面试」，查看并导出评估报告。

> 也可以不进行完整面试，直接进入「题库练习」做 Hot100 代码题。

---

## 3. 注册与登录

- 在登录页输入用户名与密码完成**注册**，随后用同一账号**登录**。
- 登录成功后，系统为你保持登录状态；访问需要登录的页面（仪表盘、面试房间、代码练习、报告、个人中心）时会自动校验。
- 若登录状态失效，系统会自动跳转回登录页，重新登录即可。

> 提示：请妥善保管账号密码。出于练习平台定位，建议不要使用与其他重要账号相同的密码。

---

## 4. 仪表盘与岗位选择

登录后进入**仪表盘**，你会看到三类岗位卡片：

| 岗位 | 考察重点 |
| ---- | -------- |
| Java 后端开发工程师 | 自动装配、JVM、并发编程等 |
| Web 前端开发工程师 | Vue/React 原理、性能优化、事件循环等 |
| Python 算法工程师 | 机器学习模型、数据结构、模型调优等 |

点击任意岗位卡片即可打开面试配置框。仪表盘也是进入代码练习、个人中心的入口。

## 5. 配置并开始一场面试

点击岗位后会弹出**面试配置框**，包含以下选项：

### 5.1 面试难度
三选一：**简单 / 中等 / 困难**（默认「中等」）。难度影响所选题目的层次。

### 5.2 正式提问轮次
通过滑块设置 **2 ~ 10 轮**（默认 6 轮）。该轮次**只统计正式知识提问**，不含开场自我介绍、简历提问与项目深挖。

### 5.3 重点考察领域（多选）
系统会根据所选岗位列出可考察的方向标签，可多选。若不选，面试官将按全流程进行；若该岗位暂无可选方向，也会自动走全流程。

### 5.4 项目深挖 GitHub 仓库（可选，最多 3 个）
- 粘贴公开的 GitHub 仓库地址（格式 `https://github.com/用户名/仓库名`）。
- 点「分析」可即时预览抓取到的仓库摘要（主语言、Star 数、描述）。
- AI 会针对你的真实项目生成定制化深挖问题。
- 可添加多个仓库（最多 3 个），也可移除。

> 抓取失败的仓库（红色提示）不会被提交。请确认仓库是公开的、地址格式正确、网络正常。

### 5.5 上传简历（可选）
通过简历上传入口提交 PDF 或文本简历，系统会做轻量结构化解析，提取技能、项目、工作年限等，注入面试官的提问上下文，使提问更贴合你的背景。

完成配置后点击「**确认配置并进入房间**」进入面试房间。

## 6. 面试房间：作答与交互

进入房间后，AI 面试官会先请你做**自我介绍**，随后按阶段推进：简历提问（如上传了简历）→ 项目深挖（如填写了仓库）→ 知识提问 → 结束语。

### 6.1 文字作答
在输入框中输入回答，发送即可。

### 6.2 语音作答
- 点击麦克风开始录音，界面显示「正在录音中…」。
- 再次操作结束录音后，系统会自动上传、用 Whisper 转写并润色文本，界面显示「AI 正在精准转录您的语音…」。
- 转写完成后作为你的回答发送，AI 面试官随即回复。

> 语音作答会额外采集你的声学特征（语速、停顿、音高、音量等），用于报告中的表达分析。**建议尽量使用语音作答**，以获得完整的表达维度评估。

### 6.3 追问与打断
- 当你的回答有深挖空间时，AI 会就同一话题**追问**（同一话题最多追问 2 次后自动切换）。
- 当回答明显离题或过短/超时，系统可能**主动打断**并引导你回到重点。

### 6.4 结束面试
- 达到设定的提问轮次后，面试官会给出结束语，面试自动结束。
- 你也可以随时点击「结束面试」提前结束。结束后系统会自动生成评估报告。

## 7. 代码练习（Hot100 ACM）

从仪表盘进入「**题库练习**」，这是独立于面试的编码练习模块，采用 ACM 模式：用完整程序从标准输入（stdin）读取数据，向标准输出（stdout）打印结果。

### 7.1 题库列表页
- 顶部统计：题目总数、已开放判题数、当前通过数。
- 筛选：可按**关键词**（题名或 slug）、**难度**、**主题标签**过滤。
- 列表项会标记题目是否已通过、是否已开放判题。前 20 题已配置完整样例与隐藏测试。
- 点「刷新题库」可重新加载。

### 7.2 题目详情与作答
点击题目进入详情页，包含题目描述、输入/输出格式、样例、约束，以及内置的**代码编辑器**（支持多语言起始模板）。当前支持 Python、Java、C++、JavaScript。

### 7.3 运行样例 vs 正式提交
- **运行样例（Run）**：仅用公开样例测试，返回每条样例的输入、期望输出、实际输出与状态，方便快速调试。运行不会记入提交历史。
- **正式提交（Submit）**：用全部测试用例（含隐藏用例）判题。隐藏用例只展示通过/失败与安全的错误信息，不暴露输入与期望输出。提交会记入历史，全部通过即标记该题为「已通过」。

### 7.4 判题结果说明
常见状态：

| 状态 | 含义 |
| ---- | ---- |
| Accepted | 全部用例通过 |
| Wrong Answer | 输出与期望不一致 |
| Compile Error | 编译失败（检查类名/语法/完整程序） |
| Time Limit Exceeded | 运行超时 |
| Runtime Error | 运行期异常 |
| Judge Error | 判题服务暂不可用，稍后重试 |

> 输出比较会忽略行尾多余空白，因此结尾换行不会导致误判。

## 8. 评估报告解读

面试结束后进入**报告页**。报告分为内容维度与表达维度，并汇总为总分。右上角「**导出 PDF**」会触发浏览器打印对话框（快捷键 Ctrl+P），在「目标」中选「另存为 PDF」即可保存。

### 8.1 内容维度（基于对话文本，由 AI 评审）

| 维度 | 含义 | 关注点 |
| ---- | ---- | ------ |
| 技术正确性 / 内容得分 | 回答在技术事实与概念上的准确程度 | 概念是否说错、关键点是否覆盖 |
| 业务场景应对 | 面对场景化问题的分析与落地能力 | 是否结合实际、是否给出可行方案 |
| 问题解决能力 | 拆解问题、给出思路与权衡的逻辑严谨性 | 思路是否清晰、是否考虑边界与取舍 |

报告还会给出**亮点（Highlights）**与**待改进点（Weaknesses）**。

### 8.2 表达维度（仅当使用语音作答时生成）

| 子维度 | 含义 | 数据来源 |
| ------ | ---- | -------- |
| 语速 | 说话快慢是否适中（每分钟词数 WPM） | 语音时长与转写词数 |
| 清晰度 | 表达是否流畅，停顿/口头禅是否过多 | 停顿比例、长停顿次数、口头禅计数 |
| 自信度 | 语气是否稳定，音量音高是否平稳 | 音高/音量的均值与波动 |

> 若全程使用文字作答，表达三维子分为 0，报告仅呈现内容维度。这是正常现象，不代表表达能力差。

### 8.3 总分与提升建议
- 总分由内容维度与表达维度加权汇总。使用语音作答时，系统会用真实语音分析结果覆盖模型对表达的粗略估计，并据此重新计算总分，使其更贴近实际表现。
- 报告末尾会针对语速、清晰度、自信度分别给出一句话建议，并提供个性化提升建议 / 学习计划。
- 如填写了 GitHub 项目，报告还会展示项目深挖相关内容。

### 8.4 如何用好报告
1. 先看总分与雷达图，定位强项与短板。
2. 再看亮点与待改进点，这是最具体的改进抓手。
3. 看表达分析建议，针对性优化语速/清晰度/自信度。
4. 按学习计划安排下一步练习方向。

## 9. 个人中心

在**个人中心**可查看与管理你的账户信息，并查看历史面试记录与成绩。历史记录列出已完成面试的岗位、难度、总分与时间，点击可回看对应报告。

## 10. 常见问题（FAQ）

- **Q：为什么我的表达分是 0？**
  A：本场全程使用了文字作答，未采集语音特征。想要表达维度评分请使用语音作答。

- **Q：语音转写不准怎么办？**
  A：系统会对转写结果做润色纠正。如仍有偏差，可在发送前改用文字补充，或在安静环境、靠近麦克风重录。

- **Q：GitHub 仓库分析失败？**
  A：确认仓库是**公开**的、地址格式为 `https://github.com/用户名/仓库名`、网络正常。失败的仓库不会被提交。

- **Q：报告多久生成？**
  A：点击「结束面试」后由系统调用 AI 评审生成，稍候即可在报告页查看。

- **Q：代码提交一直显示 Running？**
  A：判题在后台进行，稍等片刻刷新即可。若长时间无结果或显示 Judge Error，说明判题服务暂时繁忙，请稍后重试。

- **Q：能导出报告吗？**
  A：可以。在报告页点「导出 PDF」，在浏览器打印对话框中选择「另存为 PDF」。

- **Q：历史记录在哪看？**
  A：在个人中心查看所有已完成面试的评分与报告入口。
